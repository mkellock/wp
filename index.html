Thank you for the opportunity to provide a proposal for the site dedicated to Douglas Raymond Baker, a fascinating person who is certainly worthy of his story being told.

As the internet and its accompanying technologies, software, frameworks, design patterns, and end user requirements shift overtime, sometimes rapidly, the highlighted issues below are common to many aging sites today.

I hope you find this information of value and look forward to assisting in continuing Mr Baker’s story.

Analysis

A manual inspection of the site identified:

Fig 1: Douglas Raymond Baker on an iPhone 7

  • The skeuomorphic design represented by yellow and sandy backgrounds is now considered by many as a legacy design pattern. Skeuomorphic designs occupy valuable screen real estate and/or mindshare, particularly on mobile devices. Skeuomorphism offers little to help the user digest the content more productively, and although may fit the theme of the content, may ultimately detract from the messaging
  • The site lacks a responsive design, simply scaling to screen size, rather than repositioning elements such as nav and content to suit the end user’s device. See figure 1 for an example of how the site scales on an iPhone 7, the most widely used iPhone on the market1
  • The ubiquitous use of deprecated and superfluous HTML code should be avoided, including tags such as “<font>” and attributes such as “cellspacing.” These elements may be potentially unsupported in future browser versions, breaking the site’s readability and/or functionality
  • Use of tables to format the layout is semantically incorrect and can cause issues for assistive readers used by the vision impaired. Additionally, there are many missing tags that would support assistive readers
  • There is an inconsistent use of styling norms in the site’s construction, the site utilises predefined CSS stylesheets where dictated by the templated design, however, uses HTML formatting for the page content
  • Use of deprecated web design elements such as a sitemap and non-floating menu
  • Large, non-paginated and continuous bodies of text, leading to potentially poor readability
  • Although the search functionality provided by Google is excellent, it relies on the user knowing what they are looking for, additionally, menu and page navigation is done so categorically, seeming built with the assumption that the end user knows what they are looking for

Utilising automated site analysis tooling, I have identified:

  • 6,300 major bugs with the HTML
  • 1,100 minor bugs with the HTML
  • 199 code vulnerabilities
  • 3,000 major code smells

The following bugs, vulnerabilities, and “code smells” were identified as part of the automated analysis:

  • Elements and attributes deprecated in HTML5 should not be used, with 5,400 instances of deprecated elements or attributes
  • There are several CSS tags with blank file references
  • “aria-label” or “aria-labelledby” attributes should be used to differentiate similar elements to support screen reading software for the visually impaired
  • “<table>” tags should have a description, again supporting screen reading software for the visually impaired
  • Tables should have headers which supports those using assistive technologies to navigate tabular data
  • “<strong>” and “<em>” tags should be used as opposed to “<b>” tags as these convey semantic meaning rather than simply formatting text
  • “<html>” element should have a language attribute, again helping assistive technologies and search engine optimisation
  • Links with “target=_blank” should configured to prevent phishing attacks
  • Image, area, and button with image tags should have an “alt” attribute
  • “<title>” should be present in all pages
  • Links should not directly target images

Proposal

I have identified three major areas the site needs development in below:

  • Due to the enormous growth in mobile usage over the past decade, and equal decline in desktop usage2, we have seen screen sizes and dimensions vary wildly3. To meet the needs of end users that could potentially be on any screen dimension and size, the website is in dire need of a responsive design. This would reposition and scale content to best suit the device the site is being consumed on, allowing for a better experience reading its content
  • With an estimated 575,000 blind people in Australia, 70% of these above the age of 654, it is highly likely that a portion of the site’s visitors are vision impaired. A site design to scale to any device screen and work with assistive technologies like screen readers, would enable a visitor to have a much more productive visit
  • Lastly, implementing an aesthetically modern design, replacing the skeuomorphism in favour of a layout that is representative of the shifts in user tastes, whilst still being respectful of the site’s content, would help ensure readership of its content and easier navigation of the site. This can be bolstered by pagination of the site’s content, allowing users to break up their reading into smaller consumable portions
